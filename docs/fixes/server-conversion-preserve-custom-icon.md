# Server Conversion Should Preserve Custom Icon

## Status

Fixed

## Original Issue

Converting a server could replace the existing custom `server-icon.png` with the default icon written by the target platform setup.

## Root Cause

Conversion preservation already included `server-icon.png`, but restore logic only copied preserved files when the target file did not exist. The conversion/install flow creates a default icon, so the preserved custom icon was skipped.

## Solution

Updated `ConversionPreservationHelper` so preserved `server-icon.png` overwrites the generated default icon during restore, while the other preserved config files keep their previous "copy if missing" behavior.

## Files Changed

- `src/main/java/controlador/ConversionPreservationHelper.java`

## Verification

- `mvn -q -DskipTests compile` not run here because `mvn` is unavailable in this shell.
- Manual follow-up: convert a server that has a custom `server-icon.png` and verify the icon after conversion still matches the original custom image.

## Regression Notes

When conversion preservation handles files that may also be regenerated by the target platform, decide explicitly whether the preserved file or the generated file should win. `server-icon.png` should preserve the old custom asset.
